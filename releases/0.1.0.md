# Changes in JSONFold 0.1

## Initial Release

JSONFold 0.1 introduces a streaming JSON formatter that produces compact, readable output while preserving the structure of conventional pretty-printed JSON.

The formatter operates as a post-processing filter on top of existing JSON serializers and does not require parsing the full document into memory.

## Formatting Features

JSONFold combines three formatting phases:

### Scalar packing

Adjacent scalar values may be packed onto the same physical line when they fit within the configured width.

Example:

```json
[
  1, 2, 3, 4,
  5, 6, 7, 8
]
```

### Container folding

Small arrays and objects may be collapsed onto a single line.

Example:

```json
{
  "meta": { "version": 1, "ok": true }
}
```

### Folded-line joining

Adjacent folded containers may be joined together when width and nesting limits permit.

Example:

```json
[
  [ 1, 2, 3 ], [ 4, 5, 6 ]
]
```

## Presets

Built-in formatting presets provide different levels of compaction:

| Preset    | Description             |
| --------- | ----------------------- |
| `off`     | Pass-through mode       |
| `none`    | Disable all compaction  |
| `low`     | Conservative formatting |
| `med`     | Moderate formatting     |
| `default` | Balanced formatting     |
| `high`    | Aggressive formatting   |
| `max`     | Maximum compaction      |

Diagnostic presets:

* `pack`
* `fold`
* `join`

## Configuration

Formatting is controlled by:

### Width

Maximum output line width.

### Pack limits

Control how many scalar values may be packed onto a line.

* `pack_array_items`
* `pack_obj_items`
* `pack_nesting`

### Fold limits

Control which containers may be folded.

* `fold_array_items`
* `fold_obj_items`
* `fold_nesting`

### Join limits

Control when folded lines may be joined.

* `join_array_items`
* `join_obj_items`
* `join_nesting`

## Streaming architecture

JSONFold works as a streaming filter:

* buffers only currently open containers;
* streams large structures without retaining the full document;
* preserves low memory usage;
* works with existing JSON serializers.

## Language implementations

JSONFold 0.1 is available for:

* Python
* JavaScript
* Java
* Perl
* C

All implementations share the same formatting algorithm and configuration model.

## Public APIs

### Generic API

Core operations include:

* `config()`
* `format_json()`
* `write_json()`
* `create_writer()`
* `fold_text()`

### Compatibility APIs

Language-native interfaces are provided:

* Python: `dump()`, `dumps()`
* JavaScript: `stringify()`
* Java: Writer and formatter APIs
* Perl: `encode_json()`, `to_json()`
* C: streaming writer API

## Statistics

Formatting statistics are available:

* input bytes;
* output bytes;
* input lines;
* output lines.

## Command line tools

Command-line utilities are provided for supported languages.

Typical usage:

```bash
jsonfold --compact=default --width=100 < input.json
```

Features include:

* configurable width;
* preset selection;
* indentation control;
* key sorting;
* demo mode;
* file or stdin input.

## Compatibility

JSONFold preserves JSON semantics and is fully compatible with standard JSON parsers.

No changes to existing JSON producers or consumers are required.
